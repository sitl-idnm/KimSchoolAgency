# K.KIM — Design System

Фирменная дизайн-система **KIM Agency (K.KIM)** — digital-студии по разработке и
продвижению сайтов. Этот пакет позволяет верстать новые страницы в едином стиле:
смелая типографика, фирменный красный градиент, чёткая сетка, много «воздуха».

## Что в пакете
| Файл | Назначение |
|---|---|
| `kim-design-system.css` | Все токены (CSS-переменные) + готовые классы компонентов. Подключить один раз. |
| `DESIGN_SYSTEM.md` | Этот документ — принципы, токены, паттерны, примеры. |
| `CLAUDE.md` | Правила для Claude Code: как пользоваться системой при вёрстке. |
| `logo-kkim.svg` | Логотип (кольцо + буква «К» + точка). Для тёмного фона — перекрасить fill в `#fff`. |

## Подключение
```html
<link rel="stylesheet" href="kim-design-system.css">
```
Шрифты подключаются внутри CSS через `@import` (Inter Tight, Manrope, Unbounded).

## 1. Принципы дизайна
1. **Смелые заголовки, тихий текст.** Крупные плотные заголовки Inter Tight с
   отрицательным letter-spacing; основной текст — спокойный Manrope серого тона.
2. **Красный — это акцент, а не фон.** Фирменный красный/градиент только на CTA,
   иконках, одном-двух словах в заголовке, активных состояниях. Не заливать им секции.
3. **Контраст светлых и тёмных секций.** Ритм страницы задаётся чередованием белых
   секций с одной-двумя тёмными (`#16181B`) для статистики / квиза / формы.
4. **Воздух и сетка.** Много вертикального пространства (~104px между секциями),
   строгий контейнер 1440px, выравнивание по сетке.
5. **Минимум декора.** Тени почти не используются; границы тонкие (`#EDEDED`);
   скругления умеренные. Никаких лишних иконок и «слопа».
6. **Знак «✳».** Каждый блок открывается красным надзаголовком (eyebrow) со звёздочкой.

## 3. Типографика
- **Inter Tight** — заголовки, кнопки, надзаголовки, цифры. Вес 500–600, плотный
  letter-spacing (−2px у H1, −1.2px у H2).
- **Manrope** — весь основной текст, вес 400, line-height 1.6–1.65.
- **Unbounded** — только вордмарк «KIM».

| Класс | Размер | Назначение |
|---|---|---|
| `.kim-h1` | clamp(40–72px) | hero-заголовок |
| `.kim-h2` | clamp(30–46px) | заголовки секций |
| `.kim-h3` | 24px | заголовки карточек / этапов |
| `.kim-body` | 19px | основной текст |
| `.kim-small` | 15px | подписи |
| `.kim-eyebrow` | 14px uppercase | надзаголовок секции (авто «✳») |
| `.kim-stat-num` | clamp(44–68px) | большие цифры статистики |
| `.kim-step-num` | 52px | декоративный номер шага |

Акцентное слово в заголовке — обернуть в `.kim-accent` (красный).

## 4. Сетка и отступы
- Контейнер: `.kim-container` (max-width 1440px, паддинги `clamp(20px,4vw,64px)`).
- Между секциями: `.kim-section` (padding-top 104px). Тёмная секция — добавить
  `.kim-section--dark`, внутренний блок обернуть в `.kim-section-inner` (96px по вертикали).
- Радиусы: кнопки 8–10px · инпуты 10px · малые карточки 14px · карточки 20px ·
  крупные блоки 28px · круги 50%.
- Тени — почти нет. Исключение: play-кнопка `--kim-shadow-cta`.
- Типичные сетки: преимущества/CMS/статистика `repeat(3–4,1fr)` gap 20–32px;
  кейсы `repeat(2,1fr)`, акцентный кейс `grid-column:1/-1`.
- **Всегда** flex/grid + `gap` для групп элементов (не маржины на каждом).

## 5. Компоненты (классы из CSS)
- **Кнопки:** `.kim-btn` + `--primary` (градиент) / `--ghost` (контур); модификаторы
  `--sm` / `--lg`.
- **Карточки:** `.kim-card` (светлая, hover), `.kim-card--outline` (контур, hover-красный),
  `.kim-card--case` (кейс).
- **Тег:** `.kim-tag`. **Цитата:** `.kim-quote` (левый красный бордер).
- **Иконки:** `.kim-icon-circle` (соц-кнопка), `.kim-check` (чек-маркер в списках).
- **Формы:** `.kim-form-card` + `.kim-field` (на тёмном фоне).
- **Шапка:** `.kim-header` (sticky, blur). **Ссылка:** `.kim-link`. **Лого:** `.kim-logo-word`.

### Пример секции
```html
<section class="kim-section">
  <div class="kim-container">
    <span class="kim-eyebrow">Этапы разработки</span>
    <h2 class="kim-h2" style="max-width:640px;margin-bottom:52px;">
      Системный подход на каждом шаге проекта
    </h2>
    <div style="display:grid;grid-template-columns:repeat(3,1fr);gap:20px;">
      <div class="kim-card">
        <span style="font:600 18px/1 var(--kim-font-head);color:var(--kim-red);">01</span>
        <h3 class="kim-h3" style="margin-top:24px;">Аналитика и стратегия</h3>
      </div>
      <!-- … -->
    </div>
  </div>
</section>
```

### Пример CTA
```html
<a href="#consult" class="kim-btn kim-btn--primary">Обсудить проект</a>
<a href="#price"   class="kim-btn kim-btn--ghost">от 150 000 ₽</a>
```

## 6. Готовые паттерны страниц
Система рассчитана на длинные посадочные страницы. Рекомендуемый каркас:
1. Sticky-шапка (лого · навигация-якоря · соц-иконки · CTA).
2. Hero: 2 колонки — заголовок + лид + 2 кнопки · изображение (radius 24).
3. Блок «о подходе»: sticky-заголовок слева + текст/цитата справа.
4. Сетка преимуществ (карточки `.kim-card`, номер красным).
5. **Тёмная** секция статистики (крупные цифры).
6. Сетка CMS/услуг (`.kim-card--outline`).
7. Этапы — ряды с разделителями (`.kim-divider-row`).
8. Чек-лист «что входит» на фоне `--kim-bg-soft`.
9. Тёмный баннер стоимости (`--kim-grad-dark`).
10. Кейсы (`.kim-card--case`) + опционально шоурил.
11. Отзывы (`.kim-card` + звёзды + аватар).
12. **Тёмная** интерактивная секция (квиз/калькулятор).
13. FAQ-аккордеон (sticky-заголовок + раскрывающиеся вопросы).
14. **Тёмная** секция с формой заявки (`.kim-form-card`).
15. SEO-текст на `--kim-bg-soft`.
16. Тёмный футер (лого · навигация · контакты · соц).

Не обязательно использовать всё — комбинируйте блоки под задачу страницы.

## 7. Логотип
`logo-kkim.svg`: красное кольцо + чёрная буква «К» + точка. Рядом — вордмарк «KIM»
(класс `.kim-logo-word`, шрифт Unbounded). На тёмном фоне буква и точка перекрашиваются
в белый (`fill="#fff"`), кольцо остаётся красным.

---

## 8. Чего избегать
- Заливать секции сплошным красным или делать радужные градиенты-фоны.
- Использовать другие шрифты (Inter, Roboto, Arial) вместо Inter Tight / Manrope.
- Тяжёлые тени, толстые цветные рамки, скругления > 28px.
- Эмодзи в интерфейсе (фирменный акцент — символ «✳», а не эмодзи).
- «Дата-слоп»: лишние иконки, выдуманные цифры и стат-блоки без смысла.
- Мелкий текст: body не меньше 15–16px, заголовки — крупные.
