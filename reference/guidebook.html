01 — Бренд

Идентичность и позиционирование

KIM AI School — школа AI-мышления для подростков 12–17 лет. Визуальная система строится на трёх принципах: экспертность KIM.agency, технологичность AI, доверие для родителей.

Название и домен
KIM AI School ai.kim.agency

«AI» в названии выделяется красным (#cb172c) — фирменный акцент KIM.agency. Слово School в логотипе пишется с заглавной буквы, подчёркивая серьёзность проекта.

Позиционирование
🎓
Школа, не курс
Методология, артефакты, финальная защита проекта
🧠
AI-мышление, не промпты
Думать, проверять, создавать и защищать
🛡
Безопасно для родителей
Не учим списывать. Этика, данные, критическое мышление
Что НЕ мы
✗ Геймифицированный детский курс про нейросети
✗ Инфобизнес-лендинг с «результатом за 7 дней»
✗ Помощник в написании домашних заданий
✗ Кислотный, геймерский или мультяшный визуал

13 — Анимации

Все анимации сайта

Принцип: анимации усиливают смысл, не отвлекают. Нет бесполезного motion, нет тяжёлых библиотек.

Scroll Reveal
Этот блок появляется при скролле (opacity + translateY)
.reveal → opacity:0, translateY(22px) .reveal.in → opacity:1, translateY(0), transition .65s ease .reveal.d1 → transition-delay .1s .reveal.d2 → transition-delay .2s .reveal.d3 → transition-delay .3s (используется для stagger)
IntersectionObserver добавляет класс .in когда элемент входит во viewport (threshold 0.12). Каждый элемент раскрывается один раз.
AI Card — typing animation
Typing text
JS · requestAnimationFrame
Текст запроса в AI-карточке hero появляется по символам с рандомной задержкой 55–90ms. Запускается при входе hero-секции в viewport.
Blinking cursor
@keyframes blink
Мигающий курсор 0.8s step-end. Скрывается после завершения набора текста.
Check reveals
.ai-chk → .ai-chk.show
Чекмарки появляются последовательно через 600ms и 1100ms после завершения печати. opacity:0 + translateX(-6px) → opacity:1 + translateX(0).
Result fade
.ai-res → .ai-res.show
Карточка результата плавно появляется через 1800ms. opacity:0 → opacity:1, transition .5s.
Live dot pulse
@keyframes pulse-dot
Пульсирующее свечение зелёной точки «Онлайн» — box-shadow 0→5px, 2s infinite.
WebGL шейдер (ценовой блок)

Порт React-компонента animated-glassy-pricing на нативный JS+WebGL.


Как работает: GLSL фрагментный шейдер рисует три концентрические анимированные окружности с вращающимися цветами. Фон — #16181b в нормализованных значениях RGB [0.086, 0.094, 0.106].

Оптимизация: Canvas рендерится только когда секция видна на экране (IntersectionObserver). requestAnimationFrame останавливается при уходе секции из viewport.

15 — Добавить секцию

Чеклист добавления новой секции

Пошаговый процесс добавления любой новой секции в index.html без нарушения дизайн-системы.

Шаги
1
Определить тип фона
Светлая (white), серая (.sec.alt) или тёмная (.sec.dark.gbg)? Следуйте ритму чередования — не ставьте два тёмных блока подряд.
2
Выбрать тип label
Светлый фон → .lbl. Тёмный фон + AI-тема → .lbl.cy. Тёмный фон + нейтральная тема → .lbl.dim.
3
Использовать .h2 и .sub
Заголовок секции: .h2 (светлый фон) или .h2.lt (тёмный). Подзаголовок: .sub или .sub.lt.
4
Добавить .reveal к элементам
Каждый смысловой блок или карточка — добавить .reveal. Для stagger: .reveal.d1, .reveal.d2, .reveal.d3.
5
Добавить якорь id и ссылку в nav
Секция: <section id="якорь">. В nav-links: <li><a href="#якорь">Название</a></li>. В mobile nav тоже.
6
Проверить контраст на тёмном фоне
Основной текст → минимум #d1d5db. Вторичный → минимум #8d9198. Никогда не #575757 или темнее на #16181b фоне.
7
Задеплоить и обновить скриншоты
Пуш в main → GitHub Actions автоматически обновит screenshots/ через Playwright. Если нет — запустить workflow вручную.
Шаблон секции
<!-- НАЗВАНИЕ СЕКЦИИ --> <section class="sec [alt|dark gbg]" id="якорь"> <div class="wrap"> <div class="reveal"><span class="lbl [cy|dim]">МЕТКА</span></div> <div class="reveal d1"><h2 class="h2 [lt]">Заголовок</h2></div> <div class="reveal d2"><p class="sub [lt]">Подзаголовок</p></div> <!-- контент с .reveal.d1, .d2, .d3 --> </div> </section>
